# USER_TODO_CLOUD_SETUP.md — 你(开发者)需要亲自完成的事项清单

> 这些是 CC 做不了、只能你本人完成的账号注册/配置。建议在让 CC 开工前先把账号备齐,CC 需要你提供其中一些信息。

---

## 一、必须注册的账号

### 1. Cloudflare 账号(云函数 + 数据库,免费)
- 注册:https://dash.cloudflare.com/sign-up(邮箱即可,免费)
- 用途:部署激活云函数(Workers)+ 订单数据库(D1)。
- 注册后你会得到:
  - 一个 Cloudflare 账号
  - 可创建 Workers(免费10万请求/天)和 D1 数据库(免费5GB)
- **需要给 CC 的**:暂时不用给账号密码。CC 会指导你用 wrangler 命令行工具登录(浏览器授权),或你在面板里手动建好后把必要信息给它。

### 2. 自动发卡平台账号(收款 + 自动发凭证码)
- 搜索"自动发卡平台""虚拟商品发卡系统"选一家国内平台注册(如各类发卡网,注意选有正规支付通道、口碑好的)。
- 用途:挂你的软件商品,用户付款后自动发"购买凭证码"。
- 注册后配置:
  - 创建一个商品(QuantSage 激活)
  - 设置价格
  - 上传"凭证码库"(CC 会给你一个脚本批量生成凭证码,你上传上去)
- **需要确认**:该平台是否支持"卡密自动发货"(几乎都支持)。

### 3.(可选)域名
- 激活网页可先用 Cloudflare 免费的 *.workers.dev 子域,不用买域名也能上线。
- 想要更正式(如 activate.你的品牌.com),去阿里云/腾讯云买个域名(几十元/年),绑到 Cloudflare。
- MVP 阶段:**先用免费 workers.dev 子域,省这步。**

---

## 二、你需要决定/准备的信息(CC 会问你)

1. **软件定价**:一个激活码卖多少钱(如 ¥66 / ¥99)。
2. **管理员密钥(Admin-Secret)**:你自己定一个长随机字符串(如32位),用于保护永久码签发接口。**只你知道,记好。** CC 会让你把它设到云端环境变量。
3. **运维人员名单**:需要永久码的人(给他们各自的设备码,或决定用万能码)。
4. **Ed25519 密钥对**:沿用现有的(如果之前 keygen 生成过),或让 CC 重新生成一对。**私钥要交给云端(设为 Cloudflare 环境变量),公钥硬编码进客户端。**

---

## 三、私钥处理(最重要的安全事项)

- 现有的 quantsage_private.key(如果有)→ 内容设置为 Cloudflare Worker 的环境变量(Secret),**不要提交到 git、不要放进客户端安装包**。
- 公钥 → 硬编码进客户端(本来就该公开)。
- 如果重新生成密钥对:旧的已发出去的激活码会全部失效(MVP阶段还没真正卖出去,现在换正合适)。

---

## 四、上线前你要做的验收(CC 完成开发后)

1. 打开激活网页 → 输入一个测试凭证码 + 一个测试设备码 → 确认拿到激活码。
2. 把激活码贴进客户端 → 确认激活成功。
3. 同一个凭证码再用一次 → 确认被拒(一次性生效)。
4. 用 issue_permanent 脚本给自己签一个永久码 → 客户端激活 → 永久有效。
5. 发卡平台下一个测试单 → 确认自动发凭证码。

---

## 五、整体先后顺序建议

```
1. 你:注册 Cloudflare 账号
2. CC:开发云函数 + 激活网页 + 客户端改造(用占位配置)
3. 你:在 Cloudflare 建 D1 数据库、设环境变量(私钥、Admin-Secret)
4. CC:指导你用 wrangler 部署云函数,跑通
5. 你:注册发卡平台、挂商品、上传凭证码库
6. 双方:端到端测试(购买→兑换→激活)
7. 你:给运维签发10个永久码
8. 上线
```
Cloudflare 账号最先注册(CC 开发时要用)。发卡平台可以晚一点(等激活流程跑通再接)。
