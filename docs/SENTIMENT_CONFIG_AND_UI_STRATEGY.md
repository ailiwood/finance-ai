# SENTIMENT_CONFIG_AND_UI_STRATEGY.md — 情绪数据源可配置化 + 前端技术路线说明

> 放到项目 `docs/`。含两部分:① 情绪分析数据源的双通道配置(网页端 + 配置文件)② Streamlit 去留的明确结论与解耦预留。

---

## 第一部分:情绪分析数据源——双通道可配置

### 设计原则:配置双向同步,单一存储
- **一份存储,两个入口**:所有配置最终存在同一个地方(本地配置文件,如 `~/.quantsage/config.json` 或 `.env`)。
- **网页端**填的 → 写入该文件。
- **配置文件**里填的 → 网页端启动时读出来、显示在表单里。
- 两者永远一致,不存在"网页一套、文件一套"。

### 情绪数据源清单(做成可配置项,和 LLM 供应商一样的模式)
| 来源 | 类型 | 配置项 | 备注 |
|------|------|--------|------|
| 财经新闻(akshare) | 免费 | 无需 key,可开关 | 东财/新浪新闻,默认可用 |
| Finnhub | 需 key | FINNHUB_API_KEY | 美股新闻+情绪 |
| 财经API(如聚合数据/其它) | 需 key | 各自 key | 可选 |
| 社交/雪球/股吧 等 | 视来源 | 视来源 | 可选,注意 ToS |
| 自定义新闻 API | 需 key+url | CUSTOM_NEWS_URL/KEY | 兜底,任意 REST 源 |

实现要点:
1. 新建 `src/config/sentiment_sources.py`:数据源注册表(label、是否需key、key_env、url、开关默认值)。
2. 配置向导/设置页加"情绪数据源"区块:每个源一个开关 + key 输入框(密码框,只存本地)+ "测试"按钮。
3. `.env.example` 补齐各情绪源的 key 变量(留空)+ 注释说明。
4. 情绪分析模块读配置:遍历"已启用且配置完整"的源,抓取 → FinBERT/LLM 打分 → 汇总情绪指数。某源失败则跳过并告警,不阻断(降级优先)。
5. 配置读写做成统一的 `load_config()/save_config()`,网页端和文件共用,保证双向同步。

### 安全
- 所有 key 只存本地配置文件,绝不上传、绝不写日志。
- 配置文件权限提示(Windows 下放用户目录即可)。

## 第二部分:Streamlit 去留——明确结论

### 结论:V1 继续用 Streamlit,不换。

### 为什么 CC 这次的修复是正确且可接受的
- Streamlit **确实不支持 CSS `url()` 引用本地图片**,这是框架机制决定的,不是 bug。
- 正确解法就是 CC 说的:**CSS 渐变做背景 + 内嵌 SVG 做图标**(SVG 直接写成字符串内联进 HTML)。这在 Streamlit + PyInstaller 下 100% 可用,是这个生态的标准做法,不是将就。
- 所以这次修复**可以接受**,达到 V1 商用门槛(干净、专业、对比清晰)。

### 为什么现在不换框架
1. 当前真正的瓶颈是**数据准确性**,不是前端。换框架解决不了数据问题,只会拖慢进度。
2. Streamlit 配深色科技风,做到专业整洁足够 V1。之前"丑"主要来自对比度差和布局乱,正在修。
3. 换成前后端分离(FastAPI + React/Vue)是 **V2 的事**:等产品被验证有人用、需要品牌级视觉和复杂交互时再做,那时重写才不浪费。

### Streamlit 的已知天花板(留作 V2 升级触发条件)
当出现以下需求时,才考虑迁移前端:
- 需要像素级自定义 UI / 复杂动效 / 品牌级视觉。
- 需要"刷新不中断""多页复杂状态""多用户并发"等 Streamlit 偏弱的能力。
- 用户量上来,需要更强的前端性能与交互。

### 关键:本轮做"解耦预留",为 V2 铺路
即使现在用 Streamlit,也要让架构具备"未来换前端而后端不动"的能力:
1. **业务逻辑与 UI 彻底分离**:数据获取、指标计算、多智能体分析、合规审查等**全部放在 `src/core`、`src/data`、`src/analysis` 等非 UI 模块**,`src/ui`(Streamlit)只负责调用和展示,不写业务逻辑。
2. 核心能力以**函数/服务**形式暴露(输入参数、返回结构化结果),将来 React 前端可直接调同样的后端函数或包一层 FastAPI。
3. 这样 V2 换前端时,改的只是 `src/ui`,后端零改动。

> CC 实施任何 UI 改动时,顺手检查:有没有把业务逻辑误写进 Streamlit 页面?有就抽出去。
