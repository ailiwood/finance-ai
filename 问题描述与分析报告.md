## 二、根因定位：两条数据管线，一条可用一条断裂

### 2.1 双管线架构（这是问题的根源）

QuantSage 存在**两条独立的数据管线**，分别服务于不同模块：


### 2.3 根因链

`'tuple' object has no attribute 'split'` 的完整调用栈：

```
1. TA-CN Market Analyst Agent 调用 get_stock_market_data_unified(ticker="600519", ...)
2. → agents/utils/agent_utils.py:1095 → get_china_stock_data_unified(ticker, ...)
3. → dataflows/interface.py:1581 → data_source_manager.get_china_stock_data_unified(ticker, ...)
4. → dataflows/data_source_manager.py:2218 → manager.get_stock_data(symbol, ...)
5. → data_source_manager.py:1068 → result, actual_source = self._get_mongodb_data(...)
   └─ 这里 _get_mongodb_data 尝试 import app.core.database → ModuleNotFoundError
   └─ 降级到 _try_fallback_sources
6. → _try_fallback_sources 可能返回 tuple (result_str, source_name) 或单个 str
7. → get_china_stock_data_unified 拿到返回值后:
   line 2221: lines = result.split('\n')   ← 如果 result 是 tuple，这里就崩了!
```

**为什么是 tuple？** `_get_mongodb_data()` 的返回签名是 `tuple[str, str|None]`。当 MongoDB 不可用时（我们删除了 `app.core.database` 引用后它抛 ImportError），降级函数 `_try_fallback_sources` 可能因内部错误返回了 str 而非 tuple，导致调用方解包出错；或者更直接地——某处调用了返回 tuple 的函数但没有正确解包，把 tuple 当作 str 传给了 `.split()`。
